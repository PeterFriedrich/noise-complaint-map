# Phases

## Phase 1 — Static Two-Part System (MVP)

**In scope:**
- Python pipeline: fetch from Edmonton Open Data (Socrata), aggregate by day of week and neighbourhood
- Static site generated by pipeline, hosted on GitHub Pages
- Map visualization showing complaint density per day of week (7 views, toggleable)
- Basic trend indicator (are complaints up vs. same period last year?)
- Cron job on Oracle Ampere triggers pipeline on a schedule
- Pipeline commits + pushes updated `site/` to GitHub automatically

**Out of scope (deferred):**
- Isometric or 3D rendering
- Interactive filtering beyond day-of-week
- Real-time or on-demand queries
- Mobile optimization beyond basic responsiveness

**Done when:**
- [ ] Pipeline runs end-to-end on Oracle Ampere without manual intervention
- [ ] `site/data/dow-{0..6}.json` files are generated with correct schema
- [ ] GitHub Pages serves `site/index.html` and map renders without console errors
- [ ] Day-of-week toggle switches the displayed data layer
- [ ] Cron job has run successfully at least once unattended and pushed updated data
- [ ] README setup/usage section has accurate commands for running the pipeline locally
- [ ] Tests pass for transform.py aggregation logic

## Phase 2 — Isometric / 3D Rendering

**In scope:**
- Replace or augment 2D map with isometric/3D visualization (likely deck.gl HexagonLayer)
- Extruded bars or hexbins showing complaint volume by area
- Smooth day-of-week animation or transition

**Done when:**
- [ ] 3D layer renders on same data contract as phase 1 (no pipeline changes required)
- [ ] Performance acceptable on a mid-range laptop (60fps or graceful degradation)
- [ ] 2D fallback available if WebGL unavailable
